# Project AntiGravity: Master Strategy & Pillars
*This document serves as the permanent architectural reference for the Tahquitz ecosystem. Agents should reference this file if the chat context clears.*

## The Core Directives
1. **Zero Handwritten Technical Debt**: Software is purely data-driven. The entire airframe layout, skin branding variables, and hardware protocol maps are ingested at boot time from a single, centralized `config.json` schema generated by Apex.
2. **Hardware Agnosticism**: Tahquitz Core must dynamically map web-based state mutations (from Vista) into ARINC 429 binary words, TCP/IP hex commands, and CAN bus frame distributions based on the active driver profile.
3. **Automated Compliance**: Leverage the system configuration to automatically generate LaTeX-based Acceptance Test Procedures (ATP) and Configuration Status Accounting Database (CSDB) compliance files.

## The 4 Ecosystem Pillars

### 1. Tahquitz Apex (The Cloud Backend & Configurator)
* **Role**: Multi-tenant cloud-based fleet manager and visual cabin configurator.
* **Lifecycle Management**: It is a single portal for managing planes in all lifecycles. Clients log in, see "New Project" or "Current Projects".
* **Requirements Ingestion**: For a New Project, clients upload a requirements document. Apex parses it, defines what is in scope vs out of scope, generates the configuration code, and deploys it live to the aircraft.
* **Compliance**: Compiles granular, FAA/EASA-compliant ATP templates based on configured assets.

### 2. Tahquitz Core (The Local OS & Translation Layer)
* **Role**: Embedded local runtime running on aircraft server hardware.
* **Translation**: Ingests the deployment JSON at boot. Opens a WebSocket server listening to Vista. Translates UI state updates into microsecond-accurate data packets on the physical aircraft bus lines (ALTO lighting arrays, Collins routing switches).
* **Media**: Houses a stripped-down headless Jellyfin implementation delivering zero-latency streams.

### 3. Tahquitz Vista (The Luxury Passenger/Crew UI)
* **Role**: The ultra-luxury, skeuomorphic, context-aware web frontend for passengers and crew (built in Next.js).
* **Hardware-Aware**: Selectively renders widgets (Media vs Audio Player) based strictly on the `config.json` payload generated by Apex.
* **Deployments**: Runs inside featherweight Electron/Capacitor wrappers on 4.3" wall panels, 7" galley controllers, and BYOD.
* **Hot-Skinning**: Swaps luxury corporate brand assets over Starlink networks based on the charter party.

### 4. AI Testing Engine (Hardware-in-the-Loop)
* **Role**: Programmatic hardware-in-the-loop test automation engine deployed to the engineering lab test rack.
* **Execution**: Simulates UI touch points, sniffs the output bus targets to check hex line match rates, confirms video matrix carrier locks, and digitally signs off on the completed ATP compliance records.
